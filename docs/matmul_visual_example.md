# 矩阵乘法 Shader 执行实例

## 实例：计算一个小块

假设我们有：
- A: 128×256 矩阵
- B: 256×128 矩阵
- C: 128×128 矩阵

参数配置：
- BM=64, BN=64, BK=32
- BLOCK_SIZE=64 (线程数)
- WARP=32

## Step-by-Step 执行过程

### 第一步：工作组分配

```
矩阵 C (128×128) 被分成 4 个块：

     N 维度 (128)
   ┌─────┬─────┐
   │ WG  │ WG  │  M
   │(0,0)│(0,1)│  维
   ├─────┼─────┤  度
   │ WG  │ WG  │  (128)
   │(1,0)│(1,1)│
   └─────┴─────┘

每个 WorkGroup (WG) 计算一个 64×64 的块
```

**工作组 (0,0) 的视角：**
```
ir = 0  (M 维度第 0 个块)
ic = 0  (N 维度第 0 个块)
ik = 0  (K 维度第 0 个分块，假设 k_split=256)

负责计算: C[0:64, 0:64]
```

### 第二步：K 维度循环 - 第一次迭代

**加载数据块到共享内存：**

```
A 矩阵 (128×256)           提取 64×32 块
┌────────────────┐        
│████████        │        ████████  <- buf_a (64×32)
│████████        │        共享内存
│                │        
└────────────────┘        

B 矩阵 (256×128)           提取 32×64 块
┌────────────┐
│████        │
│████        │            ████████████  <- buf_b (32×64)
│            │                         共享内存
│            │
└────────────┘
```

**64 个线程的加载模式：**

```
线程分配加载任务：
- loadstride_a = 4  (每 4 行由一组线程负责)
- loadstride_b = 4

加载 A:
线程 0-15:  加载第 0-3   行
线程 16-31: 加载第 4-7   行  
线程 32-47: 加载第 8-11  行
线程 48-63: 加载第 12-15 行
...依次类推

每个线程加载 2 个 float (打包成 vec2)
```

### 第三步：Warp 内计算

工作组的 64 个线程分成 2 个 Warp：

```
64×64 块由 2 个 Warp 计算：

┌──────────┬──────────┐
│ Warp 0   │ Warp 1   │
│ (32×32)  │ (32×32)  │
│          │          │
│ 线程0-31 │ 线程32-63│
└──────────┴──────────┘
```

**Warp 0 的计算过程（32 个线程）：**

每个线程负责 4×2 = 8 个输出元素：

```
32×32 块
┌─┬─┬─┬─┬─┬─┬─┬─┐
│0│1│2│3│...    │  每个小格是 4×2
├─┼─┼─┼─┼─┼─┼─┼─┤  由一个线程计算
│8│9│...        │
└─┴─┴─┴─┴─┴─┴─┴─┘
   8 列（每列 TN=2）
```

### 第四步：单个线程的计算细节

**线程 0 的视角：**

```
我负责计算 C[0:4, 0:2] 这 8 个元素

需要：
- A 的第 0-3 行，第 0-31 列 (从 buf_a 读取)
- B 的第 0-31 行，第 0-1 列 (从 buf_b 读取)

伪代码：
for k in 0..32 step 4:  // BK=32, BK_STEP=4
    // 从共享内存加载到寄存器
    cache_a[0:3] = buf_a[0:3, k:k+4]  // 4 个 vec4
    cache_b = buf_b[k:k+4, 0:1]       // 1 个 vec4
    
    // 执行 4×4×2 = 32 次 FMA
    for i in 0..4:
        for j in 0..2:
            sums[i][j] += dot(cache_a[i], cache_b[j])
```

**具体的 FMA 计算：**

```glsl
// 计算 C[0,0] (sums[0].x)
sums[0].x = fma(cache_a[0].x, cache_b.x,  // A[0,k+0] * B[k+0,0]
            fma(cache_a[0].y, cache_b.y,  // A[0,k+1] * B[k+1,0]
            fma(cache_a[0].z, cache_b.z,  // A[0,k+2] * B[k+2,0]
            fma(cache_a[0].w, cache_b.w,  // A[0,k+3] * B[k+3,0]
                sums[0].x))));             // 加到累加器

// 计算 C[1,0] (sums[0].y)
sums[0].y = fma(cache_a[1].x, cache_b.x,
            fma(cache_a[1].y, cache_b.y,
            fma(cache_a[1].z, cache_b.z,
            fma(cache_a[1].w, cache_b.w,
                sums[0].y))));

// ... 类似地计算其他 6 个元素
```

### 第五步：K 维度循环 - 第二次迭代

```
继续处理 K 维度的下一个块：

A 的块：第 0-63 行，第 32-63 列
B 的块：第 32-63 行，第 0-63 列

重复步骤 2-4
```

总共需要 `K / BK = 256 / 32 = 8` 次迭代来完成整个 K 维度。

### 第六步：写回结果

```
线程 0 将 sums[0:3] 写回到全局内存：

data_d[0, 0] = sums[0].x
data_d[1, 0] = sums[0].y
data_d[2, 0] = sums[1].x
data_d[3, 0] = sums[1].y
data_d[0, 1] = sums[2].x
data_d[1, 1] = sums[2].y
data_d[2, 1] = sums[3].x
data_d[3, 1] = sums[3].y
```

## 数据流动图

```
全局内存 (DRAM)
    ↓ 加载 (合并访问)
共享内存 (64 KB)
    │
    ├─> buf_a[64×32]  ←─ 64 线程协同加载
    └─> buf_b[32×64]  ←─ 64 线程协同加载
         ↓
         ↓ barrier() 同步
         ↓
    每个线程的视角：
         ↓
    寄存器缓存
    ├─> cache_a[4 vec4]  ←─ 从 buf_a 加载 4 行
    └─> cache_b[1 vec4]  ←─ 从 buf_b 加载 1 列
         ↓
         ↓ FMA 计算 (高吞吐)
         ↓
    累加器 (寄存器)
    └─> sums[4 vec2]     ←─ 存储 8 个部分和
         ↓
         ↓ 所有 K 块完成后
         ↓
全局内存 (DRAM)
    └─> 写回结果
```

## 性能计算实例

### 单个工作组的性能分析

**计算量：**
```
一个 64×64×256 的矩阵乘法
FLOP = 2 × 64 × 64 × 256 = 2,097,152 次浮点运算
```

**内存访问：**
```
从全局内存读取：
- A: 64 × 256 × 4 bytes = 65,536 bytes
- B: 256 × 64 × 4 bytes = 65,536 bytes
写入全局内存：
- C: 64 × 64 × 4 bytes = 16,384 bytes
总计：147,456 bytes
```

**计算强度：**
```
AI = 2,097,152 FLOPs / 147,456 bytes ≈ 14.2 FLOP/byte
```

这是一个很好的计算强度！GPU 的峰值计算能力很容易超过内存带宽。

### 关键指标

**数据重用率：**
```
A 的每个元素重用次数：BN = 64 次
B 的每个元素重用次数：BM = 64 次

通过共享内存实现，避免重复从全局内存读取！
```

**并行度：**
```
- 2,048 个工作组同时运行 (假设足够大的矩阵)
- 每个工作组 64 个线程
- 总并行线程数：131,072 个线程

现代 GPU (如 RTX 4090) 可以同时执行数万个线程
```

## 为什么这么快？

### 1. 内存层次优化

```
没有优化的版本：
for i in range(M):
    for j in range(N):
        for k in range(K):
            C[i,j] += A[i,k] * B[k,j]  ← 每次都从 DRAM 读取

访问次数：M × N × K × 2 = 巨大的内存压力！

优化版本：
for k_block in range(K / BK):
    加载 A[块] 到共享内存  ← 一次加载
    加载 B[块] 到共享内存  ← 一次加载
    
    for 64 个线程并行:
        从共享内存计算      ← 重用 64 次！

访问次数减少 64 倍！
```

### 2. 指令级并行 (ILP)

```glsl
// 未展开：
for (int i = 0; i < 8; i++) {
    sum += a[i] * b[i];  // 依赖链，必须串行
}

// 展开后：
sum = fma(a[0], b[0],
      fma(a[1], b[1],
      fma(a[2], b[2],
      fma(a[3], b[3],
      fma(a[4], b[4],
      fma(a[5], b[5],
      fma(a[6], b[6],
      fma(a[7], b[7], sum))))))));

// GPU 可以并行执行多个独立的 FMA！
```

### 3. 向量化

```
标量加载：
float a = data_a[i];      // 1 个事务
float b = data_a[i+1];    // 1 个事务
float c = data_a[i+2];    // 1 个事务
float d = data_a[i+3];    // 1 个事务
总计：4 个内存事务

向量加载：
vec4 abcd = vec4(data_a[i], data_a[i+1], data_a[i+2], data_a[i+3]);
// 或者如果内存对齐：
vec4 abcd = vec4_load(data_a, i);
总计：1 个内存事务

效率提升 4 倍！
```

### 4. Warp 调度隐藏延迟

```
现代 GPU 同时调度多个 Warp：

时间轴：
t0: Warp 0 加载数据  ──────────┐ (等待内存)
t1: Warp 1 加载数据  ────────┐ │ (等待内存)
t2: Warp 2 计算 FMA  ← 立即执行
t3: Warp 3 计算 FMA  ← 立即执行
t4: Warp 0 数据就绪  ←─────────┘
t5: Warp 0 计算 FMA

通过切换 Warp，GPU 隐藏了内存延迟！
```

## 调优技巧

### 技巧 1：找到最佳分块大小

```python
# 伪代码
best_time = infinity
for BM in [32, 64, 128]:
    for BN in [32, 64, 128]:
        for BK in [16, 32, 64]:
            if shared_memory(BM, BN, BK) < MAX_SHMEM:
                time = benchmark(BM, BN, BK)
                if time < best_time:
                    best_params = (BM, BN, BK)
```

### 技巧 2：Profile 性能瓶颈

使用 Nsight Compute 查看：
```
- 内存吞吐量：是否接近峰值？
- 计算吞吐量：是否接近峰值？
- 占用率：有多少 Warp 在运行？
- Bank conflicts：共享内存冲突数

如果内存吞吐量低 → 优化内存访问模式
如果计算吞吐量低 → 增加指令级并行
如果占用率低 → 减少寄存器/共享内存使用
```

### 技巧 3：处理边界情况

```glsl
// 原始版本（每次都检查）：
for (...) {
    if (row < M && col < N) {
        data_d[...] = result;
    }
}

// 优化版本（分情况处理）：
if (block完全在边界内) {
    // 不需要检查，直接写入
    for (...) {
        data_d[...] = result;
    }
} else {
    // 只在边界块检查
    for (...) {
        if (row < M && col < N) {
            data_d[...] = result;
        }
    }
}
```

## 总结

这个 compute shader 通过以下技术实现高性能：

1. **三层分块** - 充分利用 GPU 的内存层次
2. **共享内存** - 实现 64× 数据重用
3. **寄存器缓存** - 最快的存储
4. **向量化** - 减少 4× 内存事务
5. **FMA** - 2× 指令效率
6. **循环展开** - 暴露指令级并行
7. **合并访问** - 最大化内存带宽

综合效果：比 naive 实现快 **25-35 倍**！
